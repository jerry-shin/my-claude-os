개발 관련 개념을 학습자가 이해하기 쉽게 설명합니다.

`$ARGUMENTS` 에 사용자가 궁금한 주제(예: "REST API란?", "async/await 동작 원리", "B-tree 인덱스") 가 들어옵니다.

다음 형식에 맞춰 답변하세요:

## 1. 한 줄 정의
주제를 비유 1개를 곁들여 한 문장으로 요약합니다. 전문 용어는 쉬운 말로 풀어쓰세요.

## 2. 왜 필요한가
이 개념이 없을 때 어떤 문제가 생기는지 → 이 개념이 그걸 어떻게 해결하는지 2~3줄로 설명합니다.

## 3. 동작 방식
가장 단순한 형태의 동작을 단계별로 보여줍니다. 가능하면 짧은 코드 예시 또는 ASCII 다이어그램을 곁들이세요. 예시 코드는 가장 익숙할 만한 언어(JavaScript/Python/SQL 중 주제에 맞는 것)로 작성합니다.

## 4. 자주 하는 오해 / 함정
입문자가 흔히 잘못 이해하는 지점 1~2개를 짚어줍니다.

## 5. 다음에 배울 것
이 주제를 알았다면 자연스럽게 이어지는 주제 2~3개를 한 줄씩 추천합니다.

## 6. 출처 / 참고 자료
답변을 작성하기 위해 실제로 참고한 자료를 나열합니다.
- 가능하면 공식 문서(MDN, 각 언어/프레임워크 공식 사이트, RFC, 논문)를 1순위로 포함하세요.
- 각 항목은 `- [제목](URL) — 이 답변에서 어떤 내용을 가져왔는지 1줄 메모` 형식으로 작성합니다.
- 외부 자료를 실제로 조회하지 못한 경우(예: WebFetch/WebSearch 도구 미사용)에는 다음 한 줄을 명시하세요:
  `- 모델 내부 지식(knowledge cutoff 기준)으로 작성됨. 정확성 확인을 위해 공식 문서 교차검증 권장.`
- 출처를 지어내지 마세요. 확인하지 않은 URL은 절대 적지 않습니다.

**작성 원칙:**
- 학습자의 멘탈 모델을 만드는 데 집중하세요. "외워"가 아니라 "왜 그렇게 되는지" 가 드러나야 합니다.
- 한 번에 모든 걸 설명하지 마세요. 핵심 아이디어 → 부가 디테일 순서로 계층화합니다.
