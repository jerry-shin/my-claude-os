Git 변경사항을 분석하고 커밋을 생성합니다.

다음 단계를 순서대로 실행하세요:

1. `git status`로 변경된 파일 목록을 확인합니다.
2. `git diff`로 실제 변경 내용을 확인합니다.
3. `git log --oneline -5`로 최근 커밋 스타일을 파악합니다.
4. 변경 내용을 분석해서 적절한 커밋 메시지를 작성합니다:
   - 첫 줄: 50자 이내의 간결한 요약 (한국어 또는 영어, 기존 커밋 스타일에 맞춤)
   - 내용이 복잡하면 빈 줄 후 상세 설명 추가
5. 커밋할 파일 목록과 커밋 메시지를 사용자에게 보여주고 확인을 요청합니다.
6. 승인되면 `git add`로 파일을 스테이징하고 `git commit`으로 커밋합니다.

**주의사항:**
- .env, 비밀키, 자격증명 파일은 절대 커밋하지 않습니다.
- 커밋 메시지 끝에 `Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>` 를 추가합니다.
- 사용자가 명시적으로 요청하지 않으면 push하지 않습니다.
