오늘 한 작업을 git log 기반으로 요약해서 일일 작업 로그를 만듭니다.

다음 단계를 순서대로 실행하세요:

1. `git log --since="today 00:00" --author="$(git config user.email)" --pretty=format:"%h %ad %s" --date=format:"%H:%M"` 명령으로 오늘 사용자가 만든 커밋 목록을 가져옵니다.
2. 커밋이 하나도 없으면 "오늘은 아직 커밋이 없습니다" 라고 알려주고 종료합니다.
3. 각 커밋의 변경 내용을 파악하기 위해 필요하면 `git show --stat <hash>` 로 파일 단위 변경 규모를 확인합니다.
4. 커밋들을 주제별로 묶어서 다음 형식의 마크다운 일일 로그를 작성합니다:

   ```
   # 일일 작업 로그 ($ARGUMENTS 또는 오늘 날짜)

   ## 요약
   - (전체 작업을 1~2줄로 요약)

   ## 작업 내역
   ### 주제 1
   - HH:MM `해시` 커밋 제목 — 한 줄 설명
   ### 주제 2
   - ...

   ## 회고 포인트
   - (배운 점 / 다음에 시도해볼 점 1~2개)
   ```

5. 작성한 로그를 사용자에게 보여주고, 파일로 저장할지 물어봅니다.
6. 저장을 원하면 `logs/YYYY-MM-DD.md` 경로에 저장합니다. (logs 디렉토리가 없으면 생성)

**주의사항:**
- `$ARGUMENTS` 가 비어있으면 오늘 날짜를 기본값으로 사용합니다.
- `$ARGUMENTS` 에 `YYYY-MM-DD` 형식의 날짜가 들어오면 그 날짜의 커밋만 조회합니다 (`--since`/`--until` 활용).
- 회고 포인트는 추측이 아니라 커밋 내용에서 실제로 드러난 패턴을 바탕으로 작성합니다.

$ARGUMENTS
